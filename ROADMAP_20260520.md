# MCP-Skeleton Roadmap

This roadmap keeps MCP-Skeleton focused on lossless context compression, exact restore, patch/replay, incremental transport, and benchmarkable large-context workflows.

## Current Position

MCP-Skeleton is now a standalone public project for AI-facing structural skeletons backed by machine-facing exact restore packages.

The current `0.1.x` line is suitable for safe dogfood use:

- compress text, one file, or one directory into `MCP-SKL.v1`
- inspect and restore text, file, directory, and incremental bundles
- run structural `apply-check` on text, file, directory, and incremental surfaces
- export and replay patches with dry-run, policy, and merge gates
- compress git-scoped incremental surfaces
- use skeleton focus modes and density modes
- filter large directories with `.mcp-skeletonignore` and repeated `--exclude`
- preserve non-UTF-8 source bytes while extracting best-effort skeleton text
- run smoke, benchmark, and dogfood self-check validation

## Recent Progress

Completed after the `v0.1.0` baseline:

- restore completeness audit that aligns original included paths, restore-package paths, restored paths, and skipped paths
- explicit skip metadata for `.git`, `__pycache__`, and `.pytest_cache`
- non-UTF-8 decode fallback for GB18030, Big5, Shift-JIS, EUC-KR, cp1252, and Latin-1
- byte-preserving restore for non-UTF-8 text files
- `.mcp-skeletonignore` and `--exclude` directory filtering
- filter metadata in `source_summary`
- preset-specific skeleton strategies and suggested excludes
- invalid input file/directory validation with non-zero CLI exit behavior
- incremental diagnostics for clean or filtered git change surfaces
- `testing/dogfood_self_check.sh` for sandboxed self-use during active development

## Safe Dogfood Policy

Use MCP-Skeleton on this repository during development, but keep it sandboxed:

- write dogfood artifacts under `testing/results/`
- restore only into ignored result directories or external temp directories
- verify restored hashes before trusting the bundle
- use `patch-apply --dry-run` before any replay write
- turn every dogfood failure into a smoke, benchmark, or dogfood regression before changing behavior

## Near-Term Priorities

### P0: Encoding And Restore Robustness

Goal: make long documents, historical sources, and Windows/macOS mixed projects safer.

- add UTF-16LE/BE and BOM-aware text decoding
- add EUC-JP and explicit GB2312 aliases where useful
- add smoke coverage for UTF-16 text-file and directory inputs
- keep restore byte-exact; encoding detection must only affect skeleton extraction
- document best-effort ambiguity when multiple legacy encodings decode the same bytes

### P0: Large Repository Stability

Goal: make token savings stronger and steadier on large projects and monorepos.

- add larger synthetic and real-ish benchmark cases for 1k, 5k, and 10k file trees
- add monorepo-style fixture generation with multiple package roots
- improve directory grouping for package/module boundaries
- add optional cache design notes before implementation
- keep benchmark outputs repeatable and machine-comparable

### P1: Semantic Skeleton Quality

Goal: improve AI-facing signal without changing restore semantics.

- add AST-backed Python symbol extraction for public classes/functions and imports
- add lightweight JS/TS import/export extraction without heavy runtime dependencies
- surface entrypoints, test files, and config files more clearly in directory skeletons
- add import graph summaries for codebase preset
- keep semantic extraction optional and failure-tolerant

### P1: CI And Dogfood Integration

Goal: make MCP-Skeleton easy to trust in continuous validation.

- document a minimal GitHub Actions workflow that runs smoke and dogfood self-check
- add a CI example for generating skeleton artifacts without uploading restore packages by default
- add a PR-change incremental example using `--base-commit`
- keep CI examples local and transparent before publishing a dedicated action

### P1: Skeleton-Layer Secret Awareness

Goal: reduce accidental leakage in AI-facing skeletons while preserving lossless restore.

- detect obvious tokens, API keys, private keys, and password-like values in skeleton text
- redact or flag only the AI-facing skeleton layer by default
- never mutate the restore package unless an explicit future policy mode says so
- add policy wording that restore packages may contain sensitive original bytes

## Later Priorities

These are useful but should wait until the CLI/API/benchmark core is steadier:

- VS Code extension for manual skeleton generation and preview
- GitHub Action marketplace packaging
- versioned skeleton catalogs for team handoff
- OpenAPI, GraphQL, and Protobuf skeleton importers
- Markdown or PlantUML export from skeletons

## Out Of Scope For Now

These can distract from the core project and should not drive near-term work:

- ecommerce, website builder, writing product, or other domain product lines
- multi-AI orchestration platforms
- long-term AI memory systems
- skeleton marketplaces
- Figma/Sketch or broad multimodal design skeletons
- enterprise access-control and audit platforms

## Stability Target

Before calling the next release broadly stable, MCP-Skeleton should have:

- smoke checks passing on macOS and Windows
- dogfood self-check passing on this repository
- byte-exact restore checks for UTF-8, non-UTF-8, UTF-16, binary, symlink, empty-dir, and filtered-directory cases
- benchmark coverage for large directory, long text, incremental, focus, density, and realistic corpus cases
- documented safe-use guidance for restore packages and patch replay
